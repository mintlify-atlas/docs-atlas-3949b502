Manitern Pro is configured through a `.env` file in the project root. This file holds all environment-specific settings including your application identity, database credentials, mail transport, and driver choices.

<Warning>
  Never commit your `.env` file to version control. It contains sensitive credentials. The repository already includes a `.gitignore` entry for `.env`. Only `.env.example` should be tracked.
</Warning>

## Environment file setup

After cloning the repository, copy the example file to create your own:

```bash
cp .env.example .env
```

Then open `.env` and update each section described below.

## Full .env.example

The following is the complete `.env.example` shipped with Manitern Pro:

```ini .env.example

LOG_CHANNEL=stack
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel
DB_USERNAME=root
DB_PASSWORD=

BROADCAST_DRIVER=log
CACHE_DRIVER=file
FILESYSTEM_DISK=local
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

MEMCACHED_HOST=127.0.0.1

REDIS_HOST=127.0.0.1
```

## APP settings

These variables control the application identity and behavior:

```ini .env
APP_NAME=Laravel
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost
```

| Variable  | Description |
|-----------|-------------|
| `APP_NAME` | The name of the application. Used in notifications and the mail `MAIL_FROM_NAME` default. |
| `APP_ENV`  | The runtime environment: `local`, `staging`, or `production`. |
| `APP_KEY`  | A 32-character base64 encryption key. Generated by `php artisan key:generate`. Must not be empty. |
| `APP_DEBUG` | When `true`, detailed error pages are shown. |
| `APP_URL`  | The full base URL of the application, used for generating links. |

<Tip>
  Set `APP_DEBUG=false` in production. Leaving it `true` exposes stack traces, environment variables, and application internals to anyone who encounters an error.
</Tip>

## Database connection

Manitern Pro uses MySQL. Update these values to match your local or production database:

```ini .env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel
DB_USERNAME=root
DB_PASSWORD=
```

| Variable | Default | Description |
|----------|---------|-------------|
| `DB_CONNECTION` | `mysql` | The database driver. Manitern Pro is built for MySQL. |
| `DB_HOST` | `127.0.0.1` | The hostname of your MySQL server. |
| `DB_PORT` | `3306` | The MySQL port. |
| `DB_DATABASE` | `laravel` | The name of the database to connect to. Create this database before running migrations. |
| `DB_USERNAME` | `root` | The MySQL user. |
| `DB_PASSWORD` | _(empty)_ | The MySQL user's password. |

## Mail settings

Manitern Pro sends mail via SMTP. The default configuration points to [Mailpit](https://github.com/axllent/mailpit) for local development:

```ini .env
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@example.com"
MAIL_FROM_NAME="${APP_NAME}"
```

For production, replace these values with your SMTP provider's credentials (e.g., Mailgun, Postmark, or Amazon SES).

## Filesystem

```ini .env
FILESYSTEM_DISK=local
```

The `FILESYSTEM_DISK` variable sets the default storage disk. The default `local` disk stores files in `storage/app`. For CV uploads to be publicly accessible via URL, the storage directory must be linked — see [Linking storage](#linking-storage) below.

## Session and cache

```ini .env
CACHE_DRIVER=file
SESSION_DRIVER=file
SESSION_LIFETIME=120
```

| Variable | Default | Description |
|----------|---------|-------------|
| `CACHE_DRIVER` | `file` | Where Laravel stores cached data. Options include `file`, `redis`, `memcached`, `array`. |
| `SESSION_DRIVER` | `file` | Where user sessions are stored. Options include `file`, `cookie`, `database`, `redis`. |
| `SESSION_LIFETIME` | `120` | Session expiry in minutes. |

## Running migrations

After configuring your database credentials, create all application tables by running:

```bash
php artisan migrate
```

This will create the following tables:

- `users` — admin accounts
- `etablissements` — partner institutions
- `stages` — internship subjects with start/end dates and status
- `services` — organisational services
- `bureaus` — bureaux within services
- `stagiaires` — trainee records linked to an etablissement, stage, and bureau
- `cvs` — CV file paths linked to stagiaires
- `absences` — absence records
- `attestations` — attestation certificate records
- `personal_access_tokens` — Laravel Sanctum tokens
- `failed_jobs` — failed queue jobs
- `password_reset_tokens` — password reset links

## Linking storage

Run the following command to create a symbolic link from `public/storage` to `storage/app/public`:

```bash
php artisan storage:link
```

<Note>
  This step is required for CV file uploads to work. Uploaded CV files are stored in `storage/app/public/cvs` and accessed via the `public/storage/cvs/` URL path. Without the storage link, CV files will not be served by the web server.
</Note>
